# Contributing to DevPane

First of all, thank you for considering contributing to DevPane.

## Getting Started

1. Fork the repository
2. Create a new branch

```bash
git checkout -b feature/my-feature
```

3. Make your changes
4. Test your changes locally
5. Commit your work

```bash
git commit -m "feat: add new feature"
```

6. Push your branch

```bash
git push origin feature/my-feature
```

7. Open a Pull Request

---

## Development Guidelines

### Code Style

- Follow existing project conventions.
- Keep functions and modules focused on a single responsibility.
- Prefer readable code over clever code.
- Avoid unnecessary dependencies.

### Commits

Use conventional commit messages when possible:

```text
feat: add workspace persistence
fix: resolve pane resize issue
docs: update configuration examples
refactor: simplify layout parser
```

### Pull Requests

A pull request should:

- Have a clear description
- Focus on a single change
- Include documentation updates when needed
- Pass all tests and checks

---

## Reporting Bugs

When opening an issue, please include:

- Operating system
- DevPane version
- Steps to reproduce
- Expected behavior
- Actual behavior

---

## Feature Requests

Feature requests are welcome.

Please explain:

- The problem you are trying to solve
- Your proposed solution
- Alternative approaches considered

---

## Questions

For questions or discussions, open an issue or start a discussion in the repository.

Thank you for helping improve DevPane.
